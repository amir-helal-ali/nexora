# واجهة Nexora الأمامية
## RFC v1.0 (مسودة)

**الحالة:** مسودة
**آخر تحديث:** 2026-07-01
**مالك الوثيقة:** هندسة منصة Nexora
**التصنيف:** داخلي — مواصفة هندسية

---

## 1. الملخص

واجهة Nexora الأمامية هي **واجهة تشغيلية حية** لـ Nexora Core (حسب الجزء 7).
مبنية بـ SvelteKit 2 + Svelte 5 + TypeScript صارم + TailwindCSS 3. تصميم
داكن أولاً مستوحى من Linear/Vercel.

## 2. المبادئ

- **إسقاط رفيع:** الواجهة إسقاط حي للنواة، لا تخزن حالة خاصة
- **الوقت الحقيقي:** كل التحديثات تأتي عبر SSE
- **الثقة الصفرية:** كل طلب يحمل رمز Bearer
- **الأداء:** تحميل أولي سريع، تفاعل سلس
- **إمكانية الوصول:** WCAG 2.1 AA

## 3. المكدس التقني

| التقنية | النسخة | الغرض |
|---------|--------|-------|
| SvelteKit | 2.x | الإطار |
| Svelte | 5.x | مكتبة UI (runes) |

## 4. الصفحات

| الصفحة | الوصف |
|--------|--------|
| `/login` | نموذج تسجيل الدخول |

| `/events` | سجل الأحداث الكامل |
| `/modules` | شبكة الوحدات |
| `/health` | صحة الأنظمة الفرعية |
| `/marketplace` | تصفح الحزم |
| `/billing` | الفواتير والاشتراكات |
| `/workflows` | سير العمل |
| `/cluster` | عقد العنقود |
| `/terminal` | طرفية NXP |
| `/api-explorer` | مستكشف API |
| `/logout` | تسجيل الخروج |

## 5. المصادقة

1. المستخدم يدخل اسم المستخدم + كلمة المرور في `/login`
2. الواجهة ترسل POST `/api/auth/login`
3. البوابة تُرجع رمز Ed25519
4. الواجهة تخزّن الرمز في `localStorage`
5. كل طلب لاحق يحمل `Authorization: Bearer <token>`
6. عند 401، الواجهة تعيد التوجيه إلى `/login`

## 6. البث المباشر

- **SSE:** `/api/core/events/stream` يبث الأحداث الحية
- **EventSource:** يُستخدم للاتصال (مع `?token=` في URL)
- **إعادة الاتصال:** تلقائية عند الانقطاع

## 7. التصميم

- **اللوحة اللونية:** داكنة أولاً (خلفية #0a0a0f، نص #e4e4e7)
- **الخط:** Inter للنص، JetBrains Mono للكود
- **التباعد:** 4/8/12/16/24/32px
- **نصف القطر:** 6px افتراضياً، 12px للبطاقات
- **الحركات:** سريعة (150ms)، تخفيف ease-out

## 8. الأداء

- **LCP:** < 1.5 ثانية
- **FID:** < 100 مللي ثانية
- **CLS:** < 0.1
- **حزمة JS الأولية:** < 100 كيلوبايت (مضغوطة)

## 9. البناء

- `npm run build` ينتج موقع ثابت في `frontend/build/`
- البوابة تخدم الأصول الثابتة في الإنتاج
- في التطوير، وكيل Vite يعيد توجيه `/api/*` إلى البوابة
